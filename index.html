Automessage Privacy Policy

Effective date: May 1, 2026

Automessage is a Chrome extension that provides AI-generated draft reply suggestions for Gmail threads. This policy explains what information the extension processes and how that information is used.

Information Automessage Processes

Automessage processes email content only when you use the extension on an open Gmail thread. To generate reply suggestions, the extension may parse the email subject, latest visible message body, Gmail thread identifier, sender name and email address when available, and your Automessage settings such as selected AI model and tone.

How Information Is Used

This information is used only to request AI-generated reply suggestions. Automessage reads the user-visible Gmail page in the browser; it does not use the Gmail API, request Gmail OAuth access, read unrelated websites, read your inbox in bulk, access email content outside the active Gmail thread, or request broad browsing permissions.

Information Shared With Services

Automessage sends the parsed Gmail context and selected settings to the configured Automessage Cloudflare Worker proxy endpoint at /v1/replies. The Worker forwards the necessary prompt content to OpenRouter so an AI model can generate reply suggestions.

Email context needed to generate suggestions is shared with the Cloudflare Worker proxy configured for this extension and with OpenRouter, which routes the request to the selected AI model provider. Cloudflare, OpenRouter, and the selected model provider may process connection metadata, such as IP address and request timing, as part of operating their services. The Worker source uses the connecting IP address only for rate limiting.

Local Storage

Automessage stores only extension settings locally in Chrome, such as selected AI model and tone preference. Reply suggestions may be cached temporarily in extension memory for performance and are not permanently stored by the extension. The extension code does not store email content, sender details, generated replies, analytics identifiers, or telemetry events in extension storage.

Analytics And Tracking

Automessage does not include analytics, advertising trackers, telemetry, or behavior tracking.

Data Sale Or Transfer

Automessage does not sell, rent, or share personal data for advertising or unrelated purposes.

Security

Automessage uses HTTPS for network requests. The OpenRouter API key is stored only in the Cloudflare Worker environment and is not included in the Chrome extension.

The extension bundle does include the configured Worker URL and a shared proxy authentication token used to call the Worker. That shared token is not an OpenRouter API key. If the token needs to be rotated, the Worker secret must be updated and the extension must be rebuilt with the new token.

Retention

The extension code does not persist email content. The Worker source does not intentionally store email content or generated replies. Email content passes through the Cloudflare Worker proxy and OpenRouter in transit. Actual logging and retention for the deployed Cloudflare Worker, Cloudflare account, OpenRouter account, and selected model provider must be confirmed by the operator of the deployed service.

Chrome Web Store Limited Use Statement

Automessage's use and transfer of information received from Chrome extension APIs and user-visible Gmail page content is limited to providing and improving the user-facing reply suggestion feature. Automessage does not use this information for advertising, does not sell this information, and does not use it to determine creditworthiness or for unrelated purposes.

Automessage does not currently use Google API Services or Gmail API OAuth scopes. If that changes, use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Your Choices

You can stop all extension processing by disabling or uninstalling Automessage. You can also clear locally stored extension settings by removing the extension from Chrome.
